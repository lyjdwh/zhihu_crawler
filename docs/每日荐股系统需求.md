# 每日股市推荐系统 - 需求文档 (v2.0)

## 一、系统目标

基于知乎股市专家的历史经验和当日分析，结合宏观经济数据，为投资者提供**具体、可执行**的每日股市操作建议。

---

## 二、数据来源

### 2.1 专家列表（可扩展）

| 专家 | 知乎ID | 特点 |
|------|--------|------|
| 奥特之父 | xu-ze-qiu | 金融界门德尔松，实战派，擅长宏观分析和中线操作 |
| MR Dang | mr-dang-77 | 独立投资人，擅长价值投资和股息策略 |

**扩展说明**：后续可增加更多专家，需要修改配置文件。

### 2.2 数据维度

#### a) 专家历史经验
- 从历史回答中提取投资理念和技巧
- 擅长分析的股票类型和行业
- 操作风格（短线/中线/长线）
- 成功的投资案例和选股逻辑

#### b) 专家当日回答
- 当日对股市行情的分析和判断
- 当日提到的具体股票/基金代码
- 当日的具体操作建议（买入/卖出/持有）
- 对后市的预测

#### c) 公司情况
- 专家提及的公司名称
- 公司当前状态（业绩、估值、走势）
- 专家对公司的具体评价

#### d) 宏观情况
- A股整体走势和成交量
- 政策动向
- 国际市场（美股、港股、汇率、大宗商品）
- 宏观经济数据（GDP、CPI等）

#### e) 个人收藏夹
- 投资理念和技巧
- 精选学习内容

### 2.3 报告生成时必须读取的数据文件

生成每日报告时，**必须按顺序读取以下所有数据文件**，缺一不可：

| 顺序 | 文件 | 用途 | 读取方式 |
|------|------|------|----------|
| 1 | `output/xu-ze-qiu_finance.json` | 奥特之父历史金融回答 | 提取投资风格、历史成功案例、长期看好的板块和个股、反复强调的投资原则 |
| 2 | `output/mr_dang_finance.json` | MR Dang历史金融回答 | 提取投资风格、持仓偏好、风险控制方法、长期投资理念 |
| 3 | `output/xu-ze-qiu_today.json` | 奥特之父当日/近期回答（爬虫产出） | 提取当日核心观点、具体操作、提到的股票代码，**必须读取所有图片** |
| 4 | `output/mr_dang_today.json` | MR Dang当日/近期回答（爬虫产出） | 提取当日核心观点、具体操作、提到的股票代码，**必须读取所有图片** |
| 5 | `output/my_collection.json`（如存在） | 个人收藏夹 | 提取投资理念和精选学习内容 |

**历史数据的作用**：
- 历史回答体现专家的**长期投资风格和偏好**，是判断当日建议可信度的重要参考
- 对比历史观点和当日观点，可以发现专家**立场变化**（如从看空转看多）
- 历史中反复提及的股票/板块代表专家的**核心持仓方向**
- 历史回答中的投资技巧和教训可作为报告的"投资理念提醒"板块内容

**注意**：历史文件可能较大，读取时重点关注最近30-50条回答即可，不需要全部精读。

### 2.4 图片内容读取要求（强制）

爬虫爬取的回答 JSON 中，每条回答的 `content` 字段包含 `[图片：路径]` 标记，`images` 字段包含图片元数据。

**必须对当日回答（xu-ze-qiu_today.json、mr_dang_today.json）中的所有图片执行以下操作**：

1. 遍历每条回答的 `images` 数组，获取每张图片的 `local_path`（已包含完整绝对路径）
2. 使用 Read 工具直接读取图片文件（路径：`local_path` 字段的值）
3. 分析图片内容，提取其中的关键信息：
   - 行情走势图 → 提取趋势方向、关键价位、技术形态
   - 财经新闻截图 → 提取标题、核心数据、政策要点
   - 官方公告/文件 → 提取关键条款、数字、日期
   - 大宗商品/指数数据 → 提取涨跌幅、价格水平
4. 将图片分析结果融入对应回答的分析中，不得忽略任何图片

**图片信息是回答内容的重要组成部分**，专家经常通过图片传递关键数据（如美联储点阵图、行情截图、新闻截图），仅读取文字内容会导致分析严重不完整。
